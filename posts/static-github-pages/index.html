Static Github Pages Deploy 하기 (GitHub Actions)

Updated: 20 May, 2025 |  at  04:27 PM

Astro와 AstroPaper를 사용해서 GithubPages에 사이트를 올릴 때 소스를 올려서 빌드를 하는 것이 아닌 빌드 완료된 dist 폴더만 publish하고 싶을 때 아래와 같은 부분을 주의해야 한다.

Github Actions 설정

githubpages에 deploy후 페이지는 뜨는데 디자인과 javascript가 반영되지 않으면 아래와 같이 HTML만 반영되고 다른 폴더가 반영이 안되어있는 경우가 많다. javascript파일이 로딩이 되지 않아 정상 동작을 하지 않는다.

commit을 분명히 한 파일인데 브라우저에서 다운로드/조회가 안됨

_astro/ClientRouter.astro_astro_type_script_index_0_lang.CtSceO8m.js 가 404 없다고 나옴

브라우저에서 동작하는 라이브러리들이 비정상 동작함

TypeError: ServiceWorker script at /~partytown/partytown-sw.js for scope
/~partytown/ encountered an error during installation.

이럴 떄는 Github Actions에서 설정을 변경하면된다. github pages 설정에서 단순 static 파일을 deploy 한다는 설정을 하면 된다. StaticSiteGenerator를 통해 생성된 dist 폴더를 deploy 하고 아래와 같는 파일을 생성해서 등록한다. .github/workflows/static.yml

GitHubPages + Google Analytics

모두 완료됐다 싶었는데 로그에 아래와 같이 계속 찍혔다. 사이트 오픈 후 브라우저 개발자 툴에서 확인해보면 쿠키 오류가 발생한다.

gtag('config', 'G-XXXXXXXXXX', {
  cookie_domain: 'none'
